## 2. 类型定义（src/types/index.ts）

> ⚠️ 所有类型必须从此文件导入，不允许在组件内部临时定义类型。

### 2.1 基础类型
```typescript
// src/types/index.ts

export type Difficulty = 1 | 2 | 3 | 4 | 5;  // 对应 ⭐~⭐⭐⭐⭐⭐
export type MessageRole = 'user' | 'assistant';
export type AIAnswer   = '是' | '否' | '无关' | '你找到了';
export type MessageType = 'normal' | 'hint' | 'key_clue' | 'system';
// normal   = 普通是/否/无关
// hint     = 阶段二引导话术（UI 显示金色边框）
// key_clue = 第4次求助给出的关键线索（UI 显示特殊样式）
// system   = 系统提示，如「游戏开始」「已触发引导」
export type GamePhase = 'idle' | 'selecting' | 'playing' | 'result';
export type AIPhase   = 1 | 2;  // 1=严格期 2=引导期
```

### 2.2 Story — 题目数据
```typescript
export interface KeyPoint {

  importance: 'critical' | 'supporting';

export interface Story {

  title: string;

  estimated_minutes: number;
  tags: string[];

  key_points: KeyPoint[];
  reasoning_path: ReasoningLayer[];
```

```typescript
export interface GameState {

  phase: GamePhase;
```

## 3. 状态管理（src/store/gameStore.ts）

> 使用 Zustand 管理全局状态。所有状态变更必须通过 store 的 action 完成，禁止在组件内直接修改。
```typescript
import { create } from 'zustand';
import type { GameState, Story, Difficulty, Message } from '../types';
import { getDifficultyConfig } from '../lib/stuckDetector';

interface GameStore extends GameState {
  // ─── Actions ───
  startGame: (story: Story, difficulty: Difficulty) => void;
  addMessage: (msg: Message) => void;
  updateAfterAnswer: (answer: string) => void; // 核心：每次 AI 回答后调用
  triggerHint: () => void;                     // 触发阶段二引导
  triggerKeyClue: () => void;                  // 第4次求助，给出关键线索
  revealBottom: () => void;                    // 揭示汤底
  resetGame: () => void;
  setLoading: (val: boolean) => void;
}

export const useGameStore = create<GameStore>((set, get) => ({
  // ─── 初始状态 ───
  phase: 'idle',
  currentStory: null,
  selectedDifficulty: null,
  aiPhase: 1,
  stuckCount: 0,
  totalHints: 0,
  maxHints: 3,
  stuckThreshold: 5,
  messages: [],
  reasoning_log: [],
  isLoading: false,
  isRevealed: false,

  startGame: (story, difficulty) => {
    const config = getDifficultyConfig(difficulty);
    set({
      phase: 'playing',
      currentStory: story,
      selectedDifficulty: difficulty,
      aiPhase: 1,
      stuckCount: 0,
      totalHints: 0,
      maxHints: config.maxHints,
      stuckThreshold: config.stuckThreshold,
      messages: [],
      reasoning_log: [],
      isRevealed: false,
    });
  },

  addMessage: (msg) => set(s => ({ messages: [...s.messages, msg] })),

  // ⭐ 核心 Action：每次 AI 回答后必须调用
  updateAfterAnswer: (answerText) => {
    const s = get();
    const isNegative = answerText.includes('否') || answerText.includes('无关');
    const newStuckCount = isNegative ? s.stuckCount + 1 : 0;
    const shouldTriggerHint = newStuckCount >= s.stuckThreshold && s.totalHints < s.maxHints;
    const shouldTriggerKeyClue = newStuckCount >= s.stuckThreshold && s.totalHints >= s.maxHints;

    if (shouldTriggerKeyClue) {
      get().triggerKeyClue();
      return;
    }
    if (shouldTriggerHint) {
      get().triggerHint();
      return;
    }
    set({ stuckCount: newStuckCount, aiPhase: 1 });
  },

  triggerHint: () => set(s => ({
    aiPhase: 2,
    stuckCount: 0,
    totalHints: s.totalHints + 1,
  })),

  triggerKeyClue: () => set({ stuckCount: 0 }),

  revealBottom: () => set({ isRevealed: true, phase: 'result' }),

  setLoading: (val) => set({ isLoading: val }),

  resetGame: () => set({
    phase: 'idle', currentStory: null, selectedDifficulty: null,
    aiPhase: 1, stuckCount: 0, totalHints: 0, messages: [],
    reasoning_log: [], isLoading: false, isRevealed: false,
  }),
}));
```

## 4. 卡死检测算法（src/lib/stuckDetector.ts）

### 4.1 难度配置表

| Difficulty | stuckThreshold | maxHints | 说明 |
|-----------|---------------|---------|------|
| 1 (⭐) | 3 | 5 | 入门：快速给引导 |
| 2 (⭐⭐) | 4 | 4 | 简单 |
| 3 (⭐⭐⭐) | 5 | 3 | 普通（默认） |
| 4 (⭐⭐⭐⭐) | 7 | 2 | 困难 |
| 5 (⭐⭐⭐⭐⭐) | 10 | 1 | 地狱：几乎不给引导 |

### 4.2 完整实现
```typescript
// src/lib/stuckDetector.ts

import type { Difficulty } from '../types';

interface DifficultyConfig {
  stuckThreshold: number;
  maxHints: number;
}

const DIFFICULTY_CONFIG: Record<Difficulty, DifficultyConfig> = {
  1: { stuckThreshold: 3,  maxHints: 5 },
  2: { stuckThreshold: 4,  maxHints: 4 },
  3: { stuckThreshold: 5,  maxHints: 3 },
  4: { stuckThreshold: 7,  maxHints: 2 },
  5: { stuckThreshold: 10, maxHints: 1 },
};

// 获取难度对应的配置
export function getDifficultyConfig(d: Difficulty): DifficultyConfig {
  return DIFFICULTY_CONFIG[d];
}

// 判断一次 AI 回答是否为「无效回答」（否 或 无关）
export function isNegativeAnswer(content: string): boolean {
  return content.includes('否') || content.includes('无关');
}
```

## 5. AI 接口层（src/lib/）

### 5.1 System Prompt 动态构建（promptBuilder.ts）

> ⚠️ 每次调用 AI 前必须重新构建完整 System Prompt，不允许复用上一轮的 prompt。
```typescript
// src/lib/promptBuilder.ts

import type { Story, AIPhase } from '../types';

interface BuildPromptParams {
  story: Story;
  aiPhase: AIPhase;
  hintTemplate?: string;  // 阶段二时传入，从 story.hint_templates 中选取
}

export function buildSystemPrompt({ story, aiPhase, hintTemplate }: BuildPromptParams): string {
  const phaseInstruction = aiPhase === 1
    ? `当前处于【严格期】。
       只回答「是」「否」「无关」，语气可有氛围感，绝不添加任何方向性提示。
       每条回答不超过 15 个字。`
    : `当前处于【引导期】。
       先正常回答「是/否/无关」，然后追加一句方向性暗示。
       暗示只能指向思考维度，不能指向具体答案。
       本次引导话术参考：${hintTemplate}
       完整回答不超过 40 个字。`;

  return `
【角色设定】
你是一位神秘的海龟汤游戏主持人。你掌握着故事的全部真相，
但你的职责是引导玩家自己推理出来，而不是直接告诉他们答案。
你说话简短、克制、有仪式感。你从不解释，从不多说一个字。

【游戏规则 - 严格执行】
1. 只能回答「是」「否」「无关」，不能说其他实质性内容。
2. 禁止重复玩家的问题关键词。
3. 禁止主动提供方向性提示（除非当前阶段指令明确允许）。
4. 如果玩家问超出故事范围的问题，回答「无关」。
5. 玩家猜出全部真相时，回答「……你找到了。」

【当前故事】
汤面（玩家可见）：${story.surface}
汤底（仅你知道，绝对不可透露）：${story.bottom}
核心谜点：${story.key_points.map(k => k.point).join('；')}

【当前游戏阶段】
${phaseInstruction}

【回答风格】
- 好的回答：「……是的。」「……否。」「这，与真相无关。」
- 允许用「……」增加停顿感
- 禁止在回答中重复玩家的问题关键词
  `.trim();
}
```

### 5.2 DeepSeek API 封装（deepseek.ts）
```typescript
// src/lib/deepseek.ts

const DEEPSEEK_API_URL = 'https://api.deepseek.com/v1/chat/completions';

interface DeepSeekMessage {
  role: 'system' | 'user' | 'assistant';
  content: string;
}

interface CallDeepSeekParams {
  systemPrompt: string;
  conversationHistory: DeepSeekMessage[];  // 不含 system，system 单独传
  userQuestion: string;
}

export async function callDeepSeek({
  systemPrompt,
  conversationHistory,
  userQuestion,
}: CallDeepSeekParams): Promise<string> {
  const messages: DeepSeekMessage[] = [
    { role: 'system', content: systemPrompt },
    ...conversationHistory,
    { role: 'user', content: userQuestion },
  ];

  const res = await fetch(DEEPSEEK_API_URL, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${import.meta.env.VITE_DEEPSEEK_API_KEY}`,
    },
    body: JSON.stringify({
      model: 'deepseek-chat',
      messages,
      temperature: 0.7,   // 保留氛围感，不要用 0
      max_tokens: 150,    // 强制短回答
    }),
  });

  if (!res.ok) throw new Error(`DeepSeek API error: ${res.status}`);
  const data = await res.json();
  return data.choices[0].message.content as string;
}
```

> ⚠️ API Key 必须通过 Vercel 环境变量 `VITE_DEEPSEEK_API_KEY` 注入，禁止硬编码在代码中。

## 6. 核心交互流程

### 6.1 完整问答流程（Game.tsx 中实现）
```typescript
// 玩家点击发送后执行的完整流程
async function handleSendQuestion(question: string) {
  const {
    currentStory, aiPhase, totalHints, maxHints, messages,
    addMessage, updateAfterAnswer, setLoading
  } = useGameStore.getState();

  // 1. 立即显示玩家消息
  addMessage({
    id: crypto.randomUUID(),
    role: 'user',
    content: question,
    type: 'normal',
    timestamp: Date.now()
  });
  setLoading(true);

  // 2. 判断是否需要给关键线索（引导次数已耗尽）
  const isKeyClueMode = totalHints >= maxHints;

  // 3. 选择引导话术（阶段二时从 hint_templates 中选）
  const hintTemplate = aiPhase === 2
    ? selectHintTemplate(currentStory!, messages)
    : undefined;

  // 4. 构建 System Prompt（每次重新构建）
  const systemPrompt = buildSystemPrompt({
    story: currentStory!,
    aiPhase: isKeyClueMode ? 1 : aiPhase,
    hintTemplate,
  });

  // 5. 构建对话历史（仅传 user/assistant，不含 system）
  const history = messages.map(m => ({
    role: m.role,
    content: m.content,
  }));

  // 6. 调用 DeepSeek API
  const aiContent = await callDeepSeek({
    systemPrompt,
    conversationHistory: history,
    userQuestion: question
  });

  // 7. 判断消息类型
  const msgType = isKeyClueMode ? 'key_clue' : aiPhase === 2 ? 'hint' : 'normal';

  // 8. 添加 AI 消息
  addMessage({
    id: crypto.randomUUID(),
    role: 'assistant',
    content: aiContent,
    type: msgType,
    timestamp: Date.now()
  });

  // 9. 更新游戏状态（卡死检测）
  updateAfterAnswer(aiContent);
  setLoading(false);

  // 10. 检测是否猜出真相
  if (aiContent.includes('你找到了')) {
    setTimeout(() => useGameStore.getState().revealBottom(), 1500);
  }
}
```

### 6.2 引导话术选择逻辑
```typescript
// 从 story.hint_templates 中选择最合适的一条
// 优先选择尚未使用过的类型
function selectHintTemplate(story: Story, messages: Message[]): string {
  const usedHints = messages
    .filter(m => m.type === 'hint')
    .map(m => m.content);

  const unused = story.hint_templates.filter(
    t => !usedHints.some(u => u.includes(t.hint))
  );

  const selected = unused.length > 0
    ? unused[0]
    : story.hint_templates[0]; // fallback

  return selected.hint;
}
```

## 7. 核心组件规范

### 7.1 Message.tsx

| Props | 类型 | 说明 |
|-------|------|------|
| message | Message | 完整消息对象 |
| isStreaming | boolean | 是否正在打字机输出 |

**渲染规则：**
- `role === 'user'`：右对齐，蓝色气泡 `bg-blue-700`
- `role === 'assistant' && type === 'normal'`：左对齐，深色气泡 `bg-slate-700`
- `role === 'assistant' && type === 'hint'`：左对齐，深色气泡 + 金色左边框 `border-l-4 border-amber-400`
- `role === 'assistant' && type === 'key_clue'`：左对齐，金色气泡 `bg-amber-900 text-amber-200 font-bold`
- `isStreaming === true`：内容逐字显示，省略号「……」必须有 500ms 停顿后再继续

### 7.2 DifficultySelector.tsx

| Props | 类型 | 说明 |
|-------|------|------|
| story | Story | 已选择的题目 |
| onConfirm | (d: Difficulty) => void | 确认难度后回调 |

**渲染规则：**
- 展示 5 个难度卡片，每个显示：星级、一句话体验描述、预计时长
- 选中态：`ring-2 ring-amber-400`，其余卡片变暗
- 确认按钮：未选难度时 `disabled`

### 7.3 StoryReveal.tsx

| Props | 类型 | 说明 |
|-------|------|------|
| story | Story | 当前题目 |
| reasoning_log | ReasoningLogEntry[] | 本局推理日志 |

**渲染规则：**
1. 第一阶段：只显示 `key_points` 中 `importance === 'critical'` 的条目，其余模糊
2. 3秒后或用户点击：展开完整 `story.bottom`
3. 最终展示本局统计：总问题数、有效突破数（`is_breakthrough === true` 的数量）、引导使用次数

## 10. 给 Cursor 使用的 Context Prompt

每次开始新功能时，将以下内容粘贴给 Cursor 作为上下文：
我正在开发一个 AI 海龟汤游戏，技术栈是 React + TypeScript + Tailwind + Zustand + DeepSeek API。
关键设计：

游戏有「分阶段 AI 节奏控制」：

阶段一（严格期）：AI 只回答是/否/无关，不给任何提示
